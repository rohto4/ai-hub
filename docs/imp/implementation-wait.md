## 暫定実装一覧（2026-03-11）

以下は仕様未確定だが、P0 を止めないために先行実装した項目です。

1. Home 一覧の DB 失敗時フォールバック
   - 現在は `/api/trends` 失敗時のみモック配列へフォールバックする。
   - 本番では DB なしでも UI 確認を止めないための暫定運用。
2. 検索結果の DB 失敗時フォールバック
   - `/api/search` 失敗時は local mock で検索を継続する。
   - 正式には API 成功時のみ検索結果を表示するかは後で判断。
3. Topic Group ボタン押下時の挙動
   - 現在は別画面遷移せず、Home 内の Topic Group セクションへスクロールし、対象記事を固定表示する。
   - 後で別ページ / ドロワー / インライン展開へ差し替える前提。
4. 300字表示の操作モデル
   - 現在はカード単位の `300字` ボタンで個別展開し、タイトル押下で外部記事を開く二段構成。
   - 最終的にカード全体タップへ寄せるかは SP/TB 実装後に再評価。
5. 保存機能
   - 現在は `localStorage` ベースで匿名保存し、サイドバーにもブラウザ依存であることを明示する。
   - Firebase ログイン連携後にサーバープロファイル同期へ差し替える。
6. 共有ポップアップ
   - `#AIHub` タグ ON/OFF、Misskey インスタンス保存、コピー導線を local state / localStorage で先行実装。
   - 投稿先 UI の最終粒度は OGP 実装と一緒に再確認する。
7. 記事から戻った際のフォーカス
   - 現在は `sessionStorage` に article id を保持し、window focus / visibilitychange で対象カードへスクロールする。
   - ブラウザ差分があるため、必要なら URL ハッシュ等へ切り替える。

### 17. Topic Group の最終遷移方式
現状:
- カード内補助導線として扱う方針までは確定
- ただし、遷移先 UI は未確定

狙い:
- 同一話題の比較導線を強化しつつ、一覧の情報密度と回遊性を落とさないこと。

リサーチ結果要約:
- `docs/mock` では Topic Group は補助導線として表現されており、`docs/spec` でも初期は補助導線前提。P0 では導線を置き、最終 UI は遷移率を見て決めるのが整合的。

候補:
- 画面遷移（別ページ `/topic/:id`）
- ドロワー/モーダル（右 or 下からスライドイン）
- インライン展開（カード下に展開）

影響:
- 一覧 UI
- Topic Group ページ設計
- 行動ログ計測

暫定運用:
- Sprint 1 は仮ページまたはプレースホルダ導線で前進可能

### 18. Topic Group ボタン押下時の暫定挙動
現状:
- 最終遷移方式が未確定なため、今の実装では暫定挙動が必要
- Topic Group 自体は仮実装先行で進める方針

狙い:
- 本実装前でもユーザー導線とログ計測を止めず、後で本仕様へ差し替えやすくすること。

リサーチ結果要約:
- 実装フェイズではホーム実データ接続が先行課題。Topic Group の本 UI を待つと進行が止まるため、仮ページや準備中導線で一度つなぐ方が全体最適。

候補:
- 仮ページへ遷移
- トースト表示で準備中
- ログのみ送信して画面変化なし

影響:
- Sprint 1 の一覧ページ実装
- `topic_group_open` ログの扱い

### 25. 管理画面の MVP 範囲
現状:
- 管理画面自体は P0 対象に含まれるが、どこまで入れるか未確定

狙い:
- 運用に必要な最小機能だけ先に入れ、公開面の開発を圧迫しないこと。

リサーチ結果要約:
- `spec` ではフィード管理・ジョブ履歴・品質監査が候補。現状は公開 UI と ingestion の立ち上げが先で、管理画面を広げすぎると Firebase 認証や Internal API が先に重くなる。

狙い:
- 運用開始に必要な最低限の管理機能を先に揃えつつ、公開面の実装を止めないこと。

リサーチ結果要約:
- 現状の ingestion / summary / cron 構成では、フィード管理だけでは運用しづらい。要約品質の確認とジョブ失敗の把握が最低限必要で、ダッシュボード一式まで広げると初動が重い。

候補:
- フィード管理のみ
- フィード管理 + ジョブ履歴
- フィード管理 + 要約監査キュー
- ダッシュボード含む一式

影響:
- Firebase 認証の優先度
- Internal API 設計
