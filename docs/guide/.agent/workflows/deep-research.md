# Command: /deep-research

## 目的 (Purpose)
作成済みのアーキテクチャ設計、実装計画、またはエラー分析レポートなどの品質を「とりあえずのAI回答レベル」から「実運用に耐えうる専門家レベル」へと昇華させます。推測に頼らず、Webのリアルなファクト（事実）で裏付けを取るための検証コマンドです。

## 実行時に自動で行われる推論とアクション (Actions)
1. **コンテキストの解析**: 現在開いているドキュメント（例: `implementation-plan.md`）や、直近のチャット履歴に記載された調査内容を読み込み、コアとなる「技術要素」や「不確実な部分（Edge cases）」を抽出する。
2. **Web検索による情報収集 (Deep Search)**:
   - 抽出した要素に対し、検索エンジン（Search Web Tool）を用いて多角的なクエリを発行する。
   - 例: 「Next.js 15 Server Actions best practices 2026」「Supabase RLS performance issues」「Tailwind v4 bento grid examples」など。
3. **ギャップの特定と補完**: 見つかった最新のベストプラクティス、既知のバグ情報、公式の推奨ツールの変更などを元のドキュメントと比較し、「不足している観点」や「時代遅れになったアプローチ」を特定する。
4. **品質向上提案の出力**: 
   - 見つかったURLのリファレンス（情報源）を提示する。
   - アーキテクチャ図や実装計画に対して、考慮すべき新しいステップやセキュリティ対策を追記・修正したものを出力（または直接ファイルを更新）する。
