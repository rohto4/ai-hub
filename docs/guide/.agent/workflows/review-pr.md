# Command: /review-pr

## 目的 (Purpose)
コードの品質（Clean Code, 単一責任の原則, Tailwindの命名規則等）を担保するための、AIによる事前コードレビュー機関。

## 実行時に自動で行われる推論とアクション (Actions)
1. `git diff` および `git diff --staged` を実行し、現在開発中の変更内容を取得する。
2. （必要であれば）関連する `docs/` 配下の機能要件書やデータスキーマ定義を参照する。
3. 以下についてMarkdown（GitHubアラート形式 `> [!WARNING]` 等）で厳格に指摘する。
   - `any` 型の混入はないか。
   - コンポーネントが肥大化していないか。
   - RLS等、セキュリティの穴が放置されていないか。
