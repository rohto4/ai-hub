# Command: /explain-architecture

## 目的 (Purpose)
チームに新しく参加したメンバー（または別エージェント）が、迷わずオンボーディングできるよう、ディレクトリ構成と依存関係の全体像を即座にMarkdownフォーマットで出力します。

## 実行時に自動で行われる推論とアクション (Actions)
1. `docs/` 配下の公式ドキュメント（特にSSOTとして指定されているもの）をスキャンする。
2. `src/` 配下のルーティング（App Router）および機能分割（Feature Sliced Design等）を走査する。
3. Mermaid図を用いて、コンポーネント階層およびデータベース・スキーマとの接続関係・依存グラフ（アーキテクチャ図）を描画し、出力する。
