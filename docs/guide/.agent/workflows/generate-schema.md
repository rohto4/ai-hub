# Command: /generate-schema

## 目的 (Purpose)
ユーザーからの生入力や、外部APIからの不確実なペイロードを安全に処理するため、コードベース全域でのバリデーション（境界防衛）を構築します。

## 実行時に自動で行われる推論とアクション (Actions)
1. 指定されたファイル（APIルートやフォームコンポーネント）の入力データの構造を解析する。
2. そのデータに必要十分な要件（文字長制限、Emailフォーマット、UUID検証、オプショナル項目の分離）を補った `z.object({})` スキーマを生成。
3. TypeScriptの型추論（`z.infer<typeof schema>`）も併せてエクスポートする形にリファクタリングする。
