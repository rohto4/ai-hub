# Command: /write-tests

## 目的 (Purpose)
ソフトウェアの「振る舞い」を保証するため、対象のクラス・関数・コンポーネントに対してのVitest, Playwright, MSWを用いた自動テストの雛形とアサーションを出力します。

## 実行時に自動で行われる推論とアクション (Actions)
1. 対象モジュールの Export されている関数・コンポーネントのシグネチャを読み取る。
2. 正常系（Happy Path）のテストケースを実装し、モックデータ（Faker.js等）を流し込む。
3. 異常系（エラーのThrow、バリデーションエラーの発生、APIフェッチ失敗等）の検証ケースを追加する。
4. `__tests__` または対象ファイルと同じ階層に `.test.ts` / `.test.tsx` ファイルとして出力し、テストの実行コマンドをユーザーに提示する。
