# セキュリティ・品質・運用仕様

最終更新: 2026-03-07

## 1. セキュリティ

1. RLS 必須
2. API 入力は Zod で検証
3. URL フェッチ先の SSRF 対策
4. Secrets はサーバー側のみ保持
5. 管理操作ログを監査可能に保持

## 2. 品質保証

1. ユニットテスト
   - 要約制約
   - 重複判定
   - スコア計算
2. 結合テスト
   - 収集->要約->表示の一連フロー
3. E2E
   - 一覧->詳細->共有

## 3. 監視

必須メトリクス:
1. 収集成功率
2. AI 解析失敗率
3. API p95
4. OGP 生成成功率
5. 新着反映遅延
6. DB 接続プール使用率（Neon Pooler 上限に対する割合）
7. インジェストキュー深度（待機ジョブ数）
8. `event_log` 書き込み遅延（バッファ滞留時間）
9. `rank_scores` 最終更新からの経過時間（陳腐化検知）

## 4. アラート

1. 15 分以上新着ゼロ（業務時間帯）
2. AI 失敗率 > 10%
3. OGP 5xx > 5%
4. バッチ連続失敗 3 回
5. DB 接続プール使用率 > 80%
6. インジェストキュー深度 > 20 件（処理詰まり検知）
7. `rank_scores` 更新停止 > 30 分（バッチ異常検知）

## 5. 運用フロー

1. 毎朝:
   - 前日品質サマリ確認
2. 毎週:
   - ランキング妥当性レビュー
3. 毎月:
   - 重み再調整と仕様更新

## 6. 障害対応

1. 検知
2. 切り分け（収集/AI/DB/配信）
3. 一時回避（フォールバック）
4. 恒久対応（再発防止）
