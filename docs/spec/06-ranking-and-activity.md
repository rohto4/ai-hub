# ランキングアルゴリズム設計

最終更新: 2026-03-05

## 1. 基本式

`TotalScore = (ExternalStats * W_e) + (SiteActivity * W_s) - (Decay * T)`

## 2. 各項目の定義

1. `ExternalStats`
   - SNS 言及、外部クリック、被参照数など
2. `SiteActivity`
   - 表示、詳細遷移、共有クリック、保存など
3. `Decay`
   - 時間経過による減衰係数
4. `T`
   - 公開からの経過時間

## 3. 初期重み案

1. `W_e = 0.55`
2. `W_s = 0.45`

備考:
- 外部偏重を避けるため、上限クリップを入れる。

## 4. 減衰関数案

1. 線形減衰（実装容易）
2. 指数減衰（自然な鮮度反映）

初期採用:
- 指数減衰（`exp(-lambda * hours)`）

## 5. 「人がいる感」演出仕様

表示案:
1. 「今この1時間で 28 回シェア」
2. 「直近 5 分で +3」
3. トピック別アクティビティパルス

実装案:
1. 5 秒間隔のポーリング
2. 可能なら SSE で push 更新
3. 数値変化時のみマイクロアニメーション

制約:
1. 実データのみ表示（擬似演出禁止）
2. 更新頻度上限を設けて負荷抑制

## 6. スコア防御設計

1. Bot 由来イベント除外
2. 同一 IP/UA の過剰イベント抑制
3. 異常値 winsorize

## 7. 監査と改善

1. 日次でトップ変動を監査
2. 係数調整ログを保持
3. KPI との相関で重み更新
