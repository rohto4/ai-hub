# 機能要件定義

最終更新: 2026-03-06

## 1. 用語

1. Genre:
   - `video`, `official`, `blog`
2. Topic Group:
   - 同一話題のクラスタ単位
3. Summary100:
   - 100文字以内の批評付き要約

## 2. 主要ユースケース

1. ユーザーが今日のトレンドを高速把握する
2. 同一トピックの動画/公式/ブログを切り替える
3. 気になるトピックを SNS へ共有する
4. 運用者が収集ソースと品質を管理する

## 3. 機能要件（必須）

## 3.1 収集

1. RSS フィード登録
2. 定期取得と差分反映
3. URL 正規化
4. 取得失敗時リトライ

受入条件:
1. 1回のジョブで取得件数/失敗件数が記録される
2. 同一 URL は再投入されない

## 3.2 解析

1. 本文抽出
2. 100文字要約生成
3. 批評生成（差分/誇大判定）
4. 難語補足タグ生成
5. 要約の可変表示モード（100/200）

受入条件:
1. 100文字超過は保存不可
2. `hype_risk` が `low|medium|high` で保存される
3. 100字を標準表示とし、200/300は展開表示で切替可能

## 3.3 重複排除

1. 同ジャンル完全一致除外（hash）
2. ジャンル横断クラスタリング

受入条件:
1. 重複記事は新規公開されない
2. Topic Group 内に最低 1 つのジャンル導線を表示

## 3.4 ランキング

1. `TotalScore = (ExternalStats * W_e) + (SiteActivity * W_s) - (Decay * T)`
2. 期間別ランキング（24h/7d/30d）
3. スコア内訳取得 API

受入条件:
1. 同条件で再計算すると同一結果
2. スコア更新履歴を保持

## 3.5 表示

1. 一覧ページ
2. 詳細ページ
3. Topic Group ページ
4. 検索/タグ絞り込み
5. 批評の折りたたみ表示（既定は非表示）
6. デバイス別情報量の出し分け（SP/TB/PC）

受入条件:
1. モバイルで片手操作可能（主要操作到達）
2. PC で 1 画面あたり情報比較しやすい密度
3. スマホは情報量を抑制し、高度機能はハンバーガー内へ退避
4. PC は分析ページ/管理ページへ直接遷移可能

## 3.6 共有

1. 記事固有 URL
2. OGP 画像生成
3. SNS プレビュー最適化

受入条件:
1. OGP に「タイトル/要約/順位/ジャンル」が入る
2. キャッシュヒット時 300ms 以内で返却

## 3.7 通知（初期リリース必須）

1. スマホ通知（PWA push）
2. 定時ダイジェスト通知（デフォルト 07:00 / 12:00 / 18:00）
3. ユーザー単位で通知 ON/OFF
4. メール通知は実装しない

受入条件:
1. 1 日 3 回の通知ジョブが実行される
2. 通知配信成功率が計測可能

## 3.8 カスタマイズ/パーソナライズ（初期最小）

1. 興味分野プリセット選択（例: Agent / Coding AI / Video AI / Safety）
2. 興味分野で絞れる（検索ワード入力の扱いは保留）
3. 匿名状態でも設定保存可能（ローカル）
4. 任意でアカウント連携し端末間同期

受入条件:
1. 未ログインでも興味分野が保存される
2. 連携時にサーバープロファイルへマージされる

## 5. 保留中の要件

1. 収集:
   - Google Alerts 中心 + 補助ソース追加の最終配分
2. 収集:
   - ユーザーに検索ワード入力を要求するかどうか
3. UI:
   - Topic Group 横断導線を最重要導線にするかどうか

## 4. 管理機能（MVP 内）

1. フィード ON/OFF
2. 収集ジョブ実行履歴
3. 低信頼クラスタ候補の確認
4. 要約品質監査キュー
