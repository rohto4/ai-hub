# フロントエンド・PWA・シェア仕様（v2）

最終更新: 2026-03-05

## 1. 画面設計

## 1.1 スマホ

1. カードスワイプ中心
2. 親指領域に主要操作（保存/共有/出典切替）
3. 1カードで「要約・順位・批評」を完結表示
4. 既定は要約100字のみ表示
5. 批評は折りたたみ（タップで展開）
6. 高度機能（分析、詳細設定）はハンバーガーメニュー配下へ

## 1.2 タブレット

1. 2 カラム構成
2. 左に一覧、右に選択詳細
3. スプリットビューで比較しやすくする

## 1.3 PC

1. 高密度テーブル + サイド詳細
2. フィルタ/タグ/期間を同時操作
3. Topic Group のソース比較を横並び表示
4. 分析ビュー/運用ビューを PC 優先表示

## 1.4 情報量出し分けルール（重要）

1. Mobile:
   - 主要 KPI のみ（順位/要約/共有）
   - 設定は最小
2. Tablet:
   - 比較操作を強化（一覧+詳細）
3. Desktop:
   - フル機能（分析、クラスタレビュー、通知統計）

## 2. PWA 要件

1. Web App Manifest
2. Service Worker（オフラインキャッシュ）
3. インストール導線（Add to Home Screen）
4. アイコン一式（maskable 含む）
5. Web Push 通知（07:00 / 12:00 / 18:00）

## 3. OGP 画像仕様

## 3.1 必須要素

1. タイトル
2. 100文字要約
3. ランキング順位
4. ジャンルアイコン
5. 日時

## 3.2 レイアウト指針

1. 1200x630 前提
2. 文字優先（背景は情報阻害しない）
3. 長文は 2-3 行で自動折返し

## 3.3 配信戦略

1. 人気記事は事前生成
2. 生成結果は CDN キャッシュ
3. 更新時は cache purge

## 3.4 OGP を主役にする UI 施策

1. 一覧カードに「共有プレビューを開く」導線を常設
2. 詳細ページ上部に OGP プレビュー固定表示
3. シェア導線クリック時に文面テンプレート提案
4. OGP 生成失敗時も要約テキスト共有を担保

## 3.5 記事サムネイル自動生成（テキスト過多対策）

1. 目的:
   - 記事一覧の視認性を上げ、テキスト塗れを防ぐ
2. 生成入力:
   - 記事タイトル
   - 100字要約
   - ジャンルタグ（video/official/blog）
3. 生成方式:
   - Claude または Gemini で画像生成
   - 生成失敗時はグラデーションのプレースホルダー
4. 利用先:
   - 一覧カードのサムネ
   - 詳細ページのヘッダ画像
   - OGP 画像合成時の背景要素
5. 運用:
   - 記事ID単位でキャッシュ
   - 不適切画像の再生成フラグを管理画面から付与可能

## 4. アクセシビリティ

1. コントラスト 4.5:1 以上
2. フォーカス可視化
3. キーボード完結
4. 動き軽減設定尊重

## 5. UI トーン（ブランド）

1. 情報密度は高く、圧迫感は低く
2. 色はジャンル識別に使い、装飾過多にしない
3. 批評表示は断定より「根拠付き示唆」を重視

## 6. 要約モード仕様

1. モード:
   - `100`（既定）
   - `200`
   - `300`
2. 切替:
   - 設定で既定値を保存
   - 記事ごとの一時切替も可能
3. レンダリング:
   - 100 はカード向け
   - 200/300 は詳細向け
