# KPI・評価設計・リリース計画

最終更新: 2026-03-05

## 1. KPI

## 1.1 利用

1. DAU / WAU
2. 一覧->詳細遷移率
3. セッションあたり閲覧トピック数

## 1.2 共有

1. 共有クリック率
2. シェア後流入率
3. OGP 表示成功率

## 1.3 品質

1. 要約監査合格率
2. 誤クラスタ率
3. 再要約率

## 2. 計測イベント設計

1. `trend_card_impression`
2. `trend_card_open`
3. `topic_switch_click`
4. `share_click`
5. `source_open`
6. `save_click`

## 3. 段階リリース

1. Phase A（内部）
   - 収集/要約/一覧の動作確認
2. Phase B（クローズドβ）
   - 共有導線、ランキング妥当性検証
3. Phase C（公開）
   - パフォーマンス最適化、運用安定化
4. Phase D（公開後改善）
   - A/B テスト基盤導入

## 4. 実験計画（A/B）※公開後

1. 要約フォーマット比較
2. カードレイアウト比較
3. ランキング重み比較

## 5. Go/No-Go 判定条件

1. 新着反映 15 分以内達成率 >= 95%
2. 共有クリック率が基準値超過
3. 要約監査合格率 >= 90%
4. 重大障害 0 件 / 7 日
