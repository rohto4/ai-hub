# AI Trend Hub 全体方針（意思決定版）

最終更新: 2026-03-06

## 1. この文書の役割

`00-overview.md` は「オーナーが最短で判断するための統合文書」とする。  
詳細は個別 spec に委譲するが、ここでは採用/保留/却下が即座に決められる粒度でまとめる。

## 2. プロダクトの北極星

AI Trend Hub は、AI 情報の「発見」「理解」「共有」を 1 画面で成立させる。

1. 発見:
   - ユーザーに検索語を要求せず、広域収集で取りこぼしを減らす。
2. 理解:
   - 100文字要約を基軸に、必要時だけ詳細を展開できる。
3. 共有:
   - OGP を主役機能に据え、貼った瞬間に価値が伝わる。

## 3. 初期リリースで固定する方針（確定）

1. 日本語特化
   - 初期は日本語 UI/要約のみ。多言語は後続。
2. ダイジェスト通知を初期搭載
   - 毎日 3 回（デフォルト 7:00 / 12:00 / 18:00）にスマホ通知。
   - メール配信は行わない。
3. 批評表示はオプション化
   - 初期表示では批評を畳み、見たい人だけ展開。
4. OGP 共有を中核機能化
   - 「おまけ」ではなく、最重要導線として設計する。
5. A/B テストは後続
   - 初期リリースでは導入しない。
6. パーソナライズはハイブリッド
   - 匿名開始 + 任意ログイン連携を採用する。
7. 要約は 100 を標準
   - 200/300 は展開モードで表示する。
8. デバイス方針
   - スマホは情報量を抑え、分析系は PC 優先にする。
9. データ活用方針
   - 行動ログを初期から収集し、通知開封率/共有率/要約モード選好を追う。

## 4. 検討中（判断待ち）

1. 収集戦略
   - Google Alerts 中心 + 補助ソース追加の最終比率を決める。
2. 検索ワード運用
   - ユーザー入力なし運用を採るか、限定的ユーザー入力を許容するかを決める。
3. Topic Group 導線の優先度
   - 主導線化するか、補助導線に留めるかを決める。

## 5. MVP スコープ（更新）

1. 収集基盤（Google Alerts + 補助ソース）※最終比率は保留
2. 正規化・重複排除・Topic Group 紐付け
3. 100文字要約 + 批評（折りたたみ表示）
4. OGP 生成と SNS 共有導線
5. 通知（7/12/18 の定時ダイジェスト）
6. ランキング（外部統計 + サイト活動 + 減衰）
7. データ収集（行動ログ）と分析基盤の最小実装

## 6. 体験設計方針（デバイス別）

1. スマホ
   - 情報過多を避ける。重要機能のみ表示。
   - 高度画面はハンバーガー配下へ。
2. タブレット
   - 一覧 + 詳細の分割表示を標準化。
3. PC
   - 情報密度を高くし、比較・分析機能を展開。
   - 一部管理/分析ページは PC 専用でも可。

## 7. 成功条件

1. 新着反映遅延: 95% が 15 分以内
2. 通知配信成功率: 98% 以上
3. 共有クリック率: 初期基準値を設定し週次改善
4. 要約監査合格率: 90% 以上
5. 月間稼働率: 99.5% 以上
