1. APIコストとレート制限の回避
- 懸念：RSSの全件処理はGemini APIの費用を圧迫する。
- 対策：`title`のみを先にLLMに投げ、「要約価値の有無」をBooleanで判定させる低コストな1次フィルタを実装。
2. ハッシュ判定の精度不足（同一トピック漏れ）
- 懸念：同一トピックでも文言が違うと別記事として並んでしまう。
- 対策：確定重複は URL / 引用元一致で扱い、類似重複は後段で pgvector または AI 類似判定を検討する。
3. モバイルでのOGP/PWAパフォーマンス
- 懸念：動的OGP生成が重いと、SNSでのシェア時にプレビュー表示が遅れる。
- 対策：生成したOGP画像は CDN または Blob Storage にキャッシュし、2回目以降は即レスポンスを返す。
4. コールドスタート・プロブレム
- 懸念：サイト公開直後、ランキングが空または固定化される。
- 対策：初期30日間は過去の「月間ベスト」的な静的データを混ぜ、賑わいを擬似的に演出する。
5. 権利関係と「引用」の成立
- 懸念：要約だけで満足され、元記事のPVを奪うとしてクレームを受ける。
- 対策：要約末尾に必ず「[元動画/記事で詳細を見る]」という目立つボタンを配置し、送客を強調する。
