# AI Trend Hub: プロジェクト憲章と技術仕様書 v1.0

## 1. プロダクトの魂（コアバリュー）
- **情報の民主化**: 専門的で追いきれないAI技術を「100文字の超要約」と「批評的視点」で、専門外の人でも5秒で理解できる形に変換する。
- **シェアの再定義**: YouTubeの直リンクを送るよりも、このサイトのリンクを送る方が「何が凄いのか」が瞬時に伝わる「インテリジェント・シェア・カード」を目指す。
- **情報の取得元を広く定義**: GoogleアラートRSSを情報収集の基準点として、なるべく広い範囲の検索ワードを設定し、後からタグ付けすることで情報の漏らしを防ぐ。

## 2. コンテンツ生成ロジック（批評的超要約）
- **100文字の構成**: 「何ができるか(事実)」+「なぜ凄いか(批評的観点)」+「誰向けか(実用性)」を凝縮。
- **批評的視点の付与**: Claude/Geminiに対し、「既存技術との差分」や「誇大広告の有無」を分析させ、単なる紹介ではない価値を付与する。
- **アクセシビリティ**: 難解な専門用語は一般的な語彙に置換、または補足タグを自動生成する。

## 3. 技術仕様：DB & パイプライン
- **DB設計**: Supabase (PostgreSQL) を使用。Notionを介さず直接投入。
- **マルチジャンル・重複排除**: 
  - `Genre_ID` (動画/公式/ブログ) ごとに記事を管理。
  - 同一トピックが複数ジャンルに存在する場合、DB上で `Topic_Group_ID` で紐付け、UI上では「動画で見る」「公式で読む」を選択可能にする。
- **スクレイピング/フェッチ**: GoogleアラートRSSからURLを抽出し、Geminiが内容を解析。ハッシュ値による同ジャンル内完全一致の自動排除。なお、AIを使用する際はAPIリソースの節約を意識。

## 4. フロントエンド：シェア・ファースト・デザイン
- **Next.js + PWA**: スマホアプリのように「ホーム画面」から1タップで最新トレンドへ。
- **究極のOGPエンジン**: 
  - `@vercel/og` により、タイトル・100文字要約・ランキング順位・ジャンルアイコンを1枚の画像に合成。
  - SNS（X, LINE, Slack）でリンクを貼った瞬間、中身が全て読める状態（パケ化）にする。
- **レスポンシブ**: PCでは情報一覧性、スマホでは親指一本で操作できるカードスワイプUI。

## 5. ランキング・アルゴリズム
- **ロジック**: $TotalScore = (ExternalStats \times W_e) + (SiteActivity \times W_s) - (Decay \times T)$
- **「人がいる感」の演出**: リアルタイムで「今この技術が〇回シェアされました」等のアクティビティを微細なアニメーションで表示。
