# mock2 導線メモ

最終更新: 2026-03-12

## 主要方針

- どの一覧画面でもカード操作の並びを揃える
- `元記事を開く` と `300文字開く` は隣接配置にする
- 強調度は `元記事を開く` と `共有` を高くし、`保存` と `後で見る` は補助操作として置く
- `類似トピックを見る` は Home / Search から Topic Group へ遷移する主導線にする
- Topic Group 内では `Feedへ戻る` をカードに置かず、画面タイトルの左下にまとめる
- Digest Center は既定で「見ると記事が減っていく」モードにする

## 画面ごとの遷移

1. Home Feed
   - `元記事を開く` → 右ペイン更新 + 遷移ログ記録
   - `300文字開く` → 右ペイン更新 + 遷移ログ記録
   - `共有` → share modal を開く
   - `保存` / `後で見る` → Saved Queue に反映
   - `類似トピックを見る` → Topic Group に遷移

2. Search Results
   - Home と同じ操作配置を維持
   - 検索から Topic Group や share modal へ直接進める

3. Topic Group
   - `Feedへ戻る` は画面上部に固定
   - 類似記事でも `保存` / `後で見る` / `元記事を開く` / `300文字開く` / `共有` の並びを統一

4. Digest Center
   - 既定モードではカード操作をした記事が一覧から消える
   - `残り続ける` モードへ切替可能
   - 日付をまたぐと既読状態をリセットする前提を表示

5. Saved Queue
   - `保存` と `後で見る` を別セクションで表示
   - どちらも同じカード操作で再訪できる

## 今回の UI ルール

- 主要 CTA:
  - `元記事を開く`
  - `共有`
- 補助 CTA:
  - `300文字開く`
  - `保存`
  - `後で見る`
- 共通カード下部レイアウト:
  - 左上段: `元記事を開く` + `300文字開く`
  - 左下段: `保存` + `後で見る`
  - 右下: `共有`
