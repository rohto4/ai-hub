# 🎨 Workflow: UX and Accessibility Review (UX/A11y レビュー)

このワークフローは、デザインやコンポーネントの骨格が完成した後に、ユーザーエクスペリエンス（UX）を最高水準へと引き上げるための包括的なチェックリストです。

## 対象 (Target)
- 全てのプレースメント、モーダル、フォーム、ナビゲーションコンポーネント

## 手順 (Steps)

1. **Fluid Typography & Bento Grid 制約チェック**
   - Tailwindのユーティリティがハードコーディングされたピクセル値になっていないか。
   - 動的な画面サイズに応答するため、要素同士のギャップ（`gap`）や余白（`p`）に一貫性はあるか。

2. **A11y (アクセシビリティ) キーボード操作の担保**
   - コンポーネント内のインタラクティブ要素（ボタン、リンク、フォーム）は、Tabキーのみでフォーカス移動できるか。
   - フォーカス時に `focus-visible:ring-2` など、視覚的なインジケータが明確に設定されているか（システムのデフォルト青枠を `outline-none` で消したままにしていないか）。

3. **Aesthetics & プレミアムプロンプト (Aesthetical Check)**
   - 色彩が純色（純白 `#FFFFFF` や純黒 `#000000`、安っぽい標準カラー）のベタ塗りになっていないか。
   - 透過色（`bg-white/10` や `backdrop-blur-md`）、繊細なボーダー、多重シャドウを用いて奥行き感（Spatial UI）を出しているか。

4. **Micro-interactions (状態変化のアニメーション)**
   - ホバー時、クリック時（`active:scale-95` 等）、フォーカス時に要素がどのように振る舞うかが定義されているか。
   - トランジション（`transition-all duration-200`）が適用され、動きが極端に唐突ではないか確認する。

5. **アーティファクトによるレビュー提出**
   - 必要に応じて、レビュー結果と改善コード（Framer Motionの追加など）をユーザーに提案する。
